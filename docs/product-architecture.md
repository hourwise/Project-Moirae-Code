# Product Architecture

This document is a documentation-only architecture pass for the current repository state on 2026-07-12. It distinguishes implemented and tested contracts from scaffolded host components, planned integrations, and proposed security posture.

## Status Terms

| Term                                   | Meaning in this repo                                                                                      |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| Implemented and tested                 | Source code exists and contract or adversarial tests exercise it in this repository.                      |
| Implemented but not integration-tested | Source code exists, but there is no end-to-end test proving the full interaction path in this repository. |
| Scaffolded                             | Code or package boundary exists, but core behavior is placeholder or stubbed.                             |
| Designed                               | Behavior is described in repository prose, but not implemented in this repository.                        |
| Proposed                               | Behavior appears only in a proposed ADR or future-facing design note.                                     |

## Process Boundaries That Are Actually Represented

- Contract boundary: `@moirae/runtime-contracts` defines shared identity, runtime, result, audit, protocol, and Moirae-specific configuration types.
- Adapter boundary: `@moirae/provider-sdk` defines the stable provider interface used by adapters.
- Supervisor boundary: `@moirae/supervisor` treats Ananke, Mnemosyne, and Horae as separate managed components and explicitly says they must not run inside the extension host.
- Extension boundary: the first-party extension is a normal VS Code extension and currently contributes only tree views, not a separate enforcement layer.

## Claims This Repository Can Support

- Shared runtime identity, health, result, audit, and protocol contracts are implemented and tested in-repo.
- Provider adapters conform to a common contract and are exercised by contract tests without live credentials.
- Supervisor health polling and crash-recovery policy are implemented and tested as library behavior.
- Skill and tool governance data models exist, including extension-policy-related types, tool manifests, and Reticle scan results.

## Claims This Repository Cannot Yet Support

- A complete governed coding loop across UI, Horae, Ananke, execution, and Mnemosyne.
- A hardened VSCodium or Electron distribution.
- Enforced host-level sandboxing for terminal, filesystem, Git, browser, extension, or credential access.
- A working approval dashboard, audit timeline, content preflight UI, or end-to-end memory workflow.
- Production secret brokering, update verification, or version-compatibility negotiation beyond declared types.

## Documentation Conflicts

### Build sequence conflict

- [docs/Moirae Code — proposed product blue.txt](./Moirae%20Code%20%E2%80%94%20proposed%20product%20blue.txt) describes a phased sequence of headless loop first, extension second, branded VSCodium third.
- [README.md](../README.md) and [docs/ROADMAP.md](./ROADMAP.md) describe Phase 1 as VSCodium shell plus IDE surfaces.
- This document does not resolve that disagreement. The roadmap and blueprint currently describe different milestone orderings.

### Package-count and scaffold-status conflict

- [README.md](../README.md) and [docs/ROADMAP.md](./ROADMAP.md) still describe the scaffold as "16 packages."
- The current tree contains 23 workspace packages with `package.json` files plus placeholder directories such as `apps/desktop-distribution`, `integrations/github`, and `integrations/ollama`.
- [packages/ui-components/src/components.ts](../packages/ui-components/src/components.ts) says actual rendering is implemented in Phase 1 webviews, while [apps/moirae-core-extension/src/extension.ts](../apps/moirae-core-extension/src/extension.ts) still exposes placeholder tree views only.
