# Upstream Dependency Matrix

This matrix tracks which Moirae Code capabilities depend on upstream repositories, what evidence is required before treating them as integrated, and what this repository currently proves.

## Status Vocabulary

| Status                     | Meaning                                                                                      |
| -------------------------- | -------------------------------------------------------------------------------------------- |
| `implemented_tested_local` | Implemented and tested in this repository without relying on the upstream runtime.           |
| `client_only_local`        | Local client or type surface exists, but the upstream capability is not integrated here.     |
| `blocked_external`         | This repository depends on upstream work that is not yet available or not yet verified here. |
| `scaffold_only_local`      | Local package boundary exists, but core behavior is placeholder or stubbed.                  |

## Notes

- The strongest verified dependency in this repository is the local shared-contract layer, not the upstream runtime integrations.
- Client packages for Horae, Ananke, and Mnemosyne are useful interface evidence, but they are not proof of integrated runtime behavior.
- No row in this matrix should be read as a claim that the headless governed loop is complete.
